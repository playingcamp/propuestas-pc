TL;DR — La regla de oro

NUNCA hagas cold outreach desde HubSpot directamente.

  • HubSpot prohíbe cold a unverified contacts (suspenden cuenta)
  • Cada cold lead suma marketing contacts = overage automático ($50-250/mes)
  • Quema deliverability de tu dominio principal (playingcamp.es)
  • Sequences HubSpot ≠ cold tools dedicados (sin warm-up, rotation, spintax)

Stack correcto: Tool cold dedicado (ReachInbox/Instantly/Smartlead) en dominio separado → solo las respuestas positivas se sincronizan a HubSpot vía Apps Script webhook.

Coste extra: ~$70-90/mes. Ahorro: dominio protegido + cuenta HubSpot a salvo + marketing contacts bajos.

1. Por qué NO usar HubSpot para cold outreach

✗ Riesgos hacer cold desde HubSpot

  • Violación TOS HubSpot: prohíben purchased lists + cold a unverified contacts. Pueden suspender cuenta sin previo aviso.
  • Marketing Contacts trap: cada cold contact que añades = facturable. 5.000 leads = salto tier inmediato +$250/mes.
  • Quema dominio playingcamp.es: quejas spam → Gmail/Outlook te marcan → emails legítimos van a spam.
  • Recovery dominio: 3-6 meses. Inviable.
  • Quota baja: 500/día Sales Pro vs ilimitado en tools cold.
  • Sin warm-up automático.
  • Sin inbox rotation.
  • Sin spintax/randomization (anti-spam).
  • Tracking básico vs Instantly/Smartlead.

✓ Ventajas stack separado

  • Dominio cold dedicado (playingcamp.io, hello-pc.com, etc.) — si quema, no afecta principal.
  • Inbox rotation: 5-10 cuentas warmed up rotando envíos.
  • Warm-up automático: tool envía/recibe entre cuentas para mejorar reputation.
  • Solo positive replies → HubSpot: contactos cualificados, no leads fríos.
  • Marketing contacts HubSpot bajos: solo leads warm = sin overages.
  • Volumen real cold: 5.000-20.000 emails/mes vs 500/día HubSpot.
  • A/B testing nativo en tools cold.
  • Compliance GDPR mejor gestionada por tools dedicados.
⚠️ Caso real: Múltiples usuarios en r/hubspot reportan cuentas suspendidas por importar listas frías. HubSpot escanea patrones y bloquea. No es leyenda urbana.

5. Código Apps Script (sync ReachInbox/Instantly → HubSpot)

Setup inicial

  1. Apps Script: script.google.com → New Project → "PC_ColdOutreach_Sync"
  2. Deploy as Web App → execute as "Me" → access "Anyone"
  3. Copia URL del deploy
  4. En ReachInbox/Instantly: Settings → Webhooks → "Reply Received" → paste URL
  5. Property Service: guarda HUBSPOT_API_KEY y HUBSPOT_PORTAL_ID

Script completo

/** * PC_ColdOutreach_Sync * Recibe webhooks de ReachInbox/Instantly * Solo crea contacto+deal en HubSpot si reply es POSITIVO * Auto-detecta vertical (mayores/empresas/partnerships) por campaign name */ const HUBSPOT_API_KEY = PropertiesService.getScriptProperties().getProperty('HUBSPOT_API_KEY'); const HUBSPOT_PORTAL_ID = PropertiesService.getScriptProperties().getProperty('HUBSPOT_PORTAL_ID'); // Mapping campaign name → HubSpot pipeline ID const PIPELINE_MAP = { 'mayores': 'pipeline_id_mayores_xxx', 'empresas': 'pipeline_id_empresas_xxx', 'partnerships': 'pipeline_id_partnerships_xxx', 'festivales': 'pipeline_id_partnerships_xxx' }; function doPost(e) { try { const data = JSON.parse(e.postData.contents); Logger.log('Webhook received: ' + JSON.stringify(data)); // Filtro 1: solo procesamos REPLY events if (data.event_type !== 'reply_received') { return ContentService.createTextOutput('ignored: not a reply'); } // Filtro 2: solo POSITIVE sentiment const positiveIntents = ['positive', 'interested', 'meeting_request']; if (!positiveIntents.includes(data.reply_sentiment)) { Logger.log('Skipped: sentiment = ' + data.reply_sentiment); return ContentService.createTextOutput('ignored: not positive'); } // Detectar vertical desde campaign name const vertical = detectVertical(data.campaign_name); const pipelineId = PIPELINE_MAP[vertical]; // 1. Crear/actualizar contacto en HubSpot const contactId = createOrUpdateContact({ email: data.lead_email, firstname: data.lead_first_name, lastname: data.lead_last_name, company: data.lead_company, lifecyclestage: 'lead', hs_lead_status: 'NEW', source: 'cold_outreach_' + data.tool_source, campaign: data.campaign_name, vertical: vertical, reply_text: data.reply_body.substring(0, 500) }); // 2. Crear deal en pipeline correcto const dealId = createDeal({ dealname: 'Cold reply: ' + data.lead_company + ' (' + vertical + ')', pipeline: pipelineId, dealstage: 'qualified_lead', amount: estimateDealValue(vertical), contactId: contactId }); // 3. Crear nota con contexto reply createNote(contactId, dealId, data); // 4. Pausar lead en campaña cold (no double-touch) pauseLeadInColdTool(data.lead_email, data.tool_source); // 5. Notificación Slack/Email a ti sendInternalNotification(data, vertical, dealId); return ContentService.createTextOutput(JSON.stringify({ status: 'success', contactId: contactId, dealId: dealId, pipeline: vertical })).setMimeType(ContentService.MimeType.JSON); } catch (err) { Logger.log('ERROR: ' + err.toString()); return ContentService.createTextOutput('error: ' + err.toString()); } } function detectVertical(campaignName) { const name = campaignName.toLowerCase(); if (name.includes('mayor') || name.includes('residencia')) return 'mayores'; if (name.includes('festival') || name.includes('conferenc')) return 'partnerships'; return 'empresas'; // default } function createOrUpdateContact(props) { const url = 'https://api.hubapi.com/crm/v3/objects/contacts/' + props.email + '?idProperty=email'; const options = { method: 'PATCH', contentType: 'application/json', headers: { 'Authorization': 'Bearer ' + HUBSPOT_API_KEY }, payload: JSON.stringify({ properties: props }), muteHttpExceptions: true }; const response = UrlFetchApp.fetch(url, options); const code = response.getResponseCode(); if (code === 404) { // Contact doesn't exist, create it return createContact(props); } return JSON.parse(response.getContentText()).id; } function createContact(props) { const url = 'https://api.hubapi.com/crm/v3/objects/contacts'; const response = UrlFetchApp.fetch(url, { method: 'POST', contentType: 'application/json', headers: { 'Authorization': 'Bearer ' + HUBSPOT_API_KEY }, payload: JSON.stringify({ properties: props }) }); return JSON.parse(response.getContentText()).id; } function createDeal(props) { const url = 'https://api.hubapi.com/crm/v3/objects/deals'; const dealResponse = UrlFetchApp.fetch(url, { method: 'POST', contentType: 'application/json', headers: { 'Authorization': 'Bearer ' + HUBSPOT_API_KEY }, payload: JSON.stringify({ properties: { dealname: props.dealname, pipeline: props.pipeline, dealstage: props.dealstage, amount: props.amount } }) }); const dealId = JSON.parse(dealResponse.getContentText()).id; // Associate contact to deal UrlFetchApp.fetch( 'https://api.hubapi.com/crm/v3/objects/deals/' + dealId + '/associations/contacts/' + props.contactId + '/deal_to_contact', { method: 'PUT', headers: { 'Authorization': 'Bearer ' + HUBSPOT_API_KEY } } ); return dealId; } function createNote(contactId, dealId, data) { const noteBody = 'COLD REPLY DETECTED\n\n' + 'Campaign: ' + data.campaign_name + '\n' + 'Sentiment: ' + data.reply_sentiment + '\n' + 'Tool: ' + data.tool_source + '\n\n' + '--- REPLY ---\n' + data.reply_body; UrlFetchApp.fetch('https://api.hubapi.com/crm/v3/objects/notes', { method: 'POST', contentType: 'application/json', headers: { 'Authorization': 'Bearer ' + HUBSPOT_API_KEY }, payload: JSON.stringify({ properties: { hs_note_body: noteBody, hs_timestamp: Date.now() }, associations: [ { to: { id: contactId }, types: [{ associationCategory: 'HUBSPOT_DEFINED', associationTypeId: 202 }] }, { to: { id: dealId }, types: [{ associationCategory: 'HUBSPOT_DEFINED', associationTypeId: 214 }] } ] }) }); } function pauseLeadInColdTool(email, tool) { // Llamada API a Instantly/Smartlead/ReachInbox para pausar lead // Cada tool tiene endpoint diferente — implementar según tu tool if (tool === 'instantly') { const apiKey = PropertiesService.getScriptProperties().getProperty('INSTANTLY_API_KEY'); UrlFetchApp.fetch('https://api.instantly.ai/api/v1/lead/update', { method: 'POST', contentType: 'application/json', payload: JSON.stringify({ api_key: apiKey, email: email, status: 'paused' }) }); } } function estimateDealValue(vertical) { const values = { 'mayores': 1500, 'empresas': 2500, 'partnerships': 800 }; return values[vertical] || 1000; } function sendInternalNotification(data, vertical, dealId) { GmailApp.sendEmail( 'hello@playingcamp.com', '🔥 Cold reply: ' + data.lead_company + ' (' + vertical + ')', 'Lead respondió positivo en campaña cold.\n\n' + 'Email: ' + data.lead_email + '\n' + 'Vertical: ' + vertical + '\n' + 'Reply: ' + data.reply_body.substring(0, 300) + '\n\n' + 'Ver deal: https://app.hubspot.com/contacts/' + HUBSPOT_PORTAL_ID + '/deal/' + dealId ); }

8. Errores comunes a evitar

  • NO uses dominio principal para cold. Siempre dominio secundario.
  • NO sincronices TODOS los replies a HubSpot. Solo positive sentiment. Otherwise inflas marketing contacts.
  • NO importes 10.000 leads de golpe en Instantly. Empieza 500-1.000 para no quemar dominio nuevo.
  • NO saltes warm-up 14 días. Es crítico. Sin warm-up → spam folder.
  • NO uses mismo template para los 3 verticales. Personaliza por industria.
  • NO ignores SPF/DKIM/DMARC. Sin esto, emails van a spam.
  • NO envíes >100/día/inbox al principio. Sube gradualmente.
  • NO uses Apps Script trigger time-based para cold. Solo webhook reactivo a replies.
